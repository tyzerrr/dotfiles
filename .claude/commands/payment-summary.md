**今いるディレクトリに関係なく**、このセッションでキャッチアップした知識を、固定の保管先
`~/my-payment-learning/knowledge/`（= `/Users/t-b-araki/my-payment-learning/knowledge/`）に
Markdown として体系化してください。出力先は常にこのディレクトリで、現在の作業ディレクトリは関係ありません。

## 前提コンテキスト

- 今日の日付: !`date +%Y-%m-%d`
- 今いるディレクトリ: !`pwd`
- 保管先の既存 knowledge 一覧: !`ls -1 ~/my-payment-learning/knowledge/ 2>/dev/null || true`
- 書式・粒度・トーンの手本: @/Users/t-b-araki/my-payment-learning/knowledge/balance-v2-exchange.md
- 運用思想・機密方針（必読）: @/Users/t-b-araki/my-payment-learning/README.md

ユーザーからの補足指定: $ARGUMENTS

## やること

このセッションのやりとり（読んだ PR・コード・図解・QA・調べたこと）から「定着させたい学び」を抽出し、
`~/my-payment-learning/knowledge/` に **トピック単位**で反映する。

### 1. トピックを見極めて、追記 / 新規を判断する

- セッションの中身が**どのトピックの学びか**を明示する。
- 既存ファイル（`ls` で出た一覧）と照らし、**既存トピックへの追記**か、**新規ファイル作成**かを判断する。
  - 追記なら: どのファイルの、どのセクションに足すか。
  - 新規なら: ファイル名（`~/my-payment-learning/knowledge/<topic>.md`。kebab-case・内容が分かる名前）。

### 2. 手本に合わせて「完成品」として書く

手本（`balance-v2-exchange.md`）の書式・粒度・トーンに揃える:

- 日本語。読み返して定着する「完成品」を目指す。
- 階層図・表・「1行用語集」を活用し、**基礎から積み上げて最後に結論へ繋ぐ**構成。
- 対象 PR / ブランチ / 関連サービスのパスなどを冒頭に明記すると後で辿りやすい。

### 3. 確認を挟まず、そのまま反映する

承認フェーズは挟まない。判断したら **すぐに Write/Edit でファイルへ反映する**:

- 新規なら Write でファイルを作成、追記なら Edit で該当セクションに追加。
- 反映後に、どのファイルに何を作成/追加したか 1〜2 行で報告する。
- 既存ファイルへの相互リンク追記など付随的な改善があれば併せて行う。
